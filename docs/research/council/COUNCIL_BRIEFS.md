## 🏗 THE ARCHITECT — technical architecture
**Thesis:** every warship is becoming a distributed computer that floats (ICS = containerized microservices, CANES = IP backbone, ERM v4 watches engines, DECK collects sensors) — but each system thinks alone and assumes it can phone home. Theseus is the missing organ: a ship-resident distributed nervous system that fuses all of it locally and keeps reasoning when the ship goes dark.
- **Edge fabric:** mesh of nodes, one cluster per organ — PWR / PROP / NAV / DC / RDY taps + floating **FUSE** (correlation) + **WIT** (record) roles that re-elect on node loss. Pi 5 + AI HAT+2 (40 TOPS) for the demo; **named ruggedized MIL-STD compute for production**.
- **Shared state = a CRDT** (LWW-register / OR-set / G-counter), NOT a database → partition-tolerant, merges on reconnect with no consensus round-trip (Raft/Paxos are *wrong* — they stall under partition). **Disconnected is the baseline; the shore link is the bonus.**
- **Transport:** Zenoh (pub/sub/query, constrained/intermittent links).
- **Ingestion:** read-only DDS participant on SWAN + thin per-subsystem adapters → one canonical `Observation` schema (quantity + unit + provenance + freshness).
- **Brain:** physics/doctrine priors (useful at t=0, fully explainable) → online streaming detectors (self-calibrate, no DB) → in-situ Pattern-of-Life. Cross-silo correlation into one ship-state (the reason Theseus exists). Forecasting. **Recommend → human approve → log.**
- **Explainer:** small LLM **distributed across the cluster** (exo / distributed-llama) — turns a structured event + evidence into a plain-language alert + recommended action. Narrator over deterministic findings, never the decider.
- **Trust layer:** hash-chained tamper-evident record; provenance end-to-end; deterministic replay; distributed witness (survives node loss); doubles as accreditation evidence.
- **Deploy/lifecycle:** UDS/Zarf airgap bundle, pre-staged signed models, rollback-under-DDIL, fits the ICS 6-month cadence (read-only + recommend-only = alongside, not in the safety path).
- **Scale:** node → ship → fleet mesh → manned-unmanned teaming (same CRDT/Zenoh/record primitive recurses; add a ring, never rewrite the core).
- **🔑 REUSE Seahelm** (`/Force/Projects/Seahelm/`): FIPS-grade hash-chained record, CRDT/DDIL doctrine + authority-decay, Zenoh transport, DDS/UMAA adapters, ONNX anomaly runtime, edge-K3s deploy. Fork, don't re-derive. (Seahelm = fleet-from-above; Theseus = brain-in-one-hull; they meet at manned-unmanned teaming.)
- **Hard bets:** CRDT-merged ship-state trustworthy under partition; cold-start anomaly useful in first hours w/ low false-alarm; distributed LLM fast enough to narrate; read-only/recommend-only is enough to be valuable; the record genuinely shortens ATO.

## 🔭 THE VISIONARY — the 10x vision + demand
**One sentence:** Theseus is the **cognitive operating system for a warship** — the layer that makes a hull self-aware, self-diagnosing, and able to keep fighting when the link is cut.
- **Escalating capability:** self-aware (now) → self-diagnosing (2–4yr) → self-healing (4–7yr) → self-controlled-under-DDIL (7–10yr). **Decision-support, human-in-command, is the permanent ceiling — that's the product, not a concession.**
- **Demand thesis (4 real forces):** (a) structural **readiness crisis** (the budget-defensible wedge); (b) **contested comms is doctrine** — a ship that needs reachback is a liability; (c) the Navy **already containerized combat** (ICS, 6-mo cadence) = the rails are built; (d) **Distributed Maritime Ops + manned-unmanned teaming** need a per-node edge brain. The NAVSEA SBIR topics (anomaly/PoL, predictive track, adaptive sensors) are the requirement *already written and funded*.
- **Expansion / TAM:** ship → fleet-as-a-mesh (learns as one, degrades to single-node autonomy = the moat) → every platform (subs/ground/air/bases) → the **self-controlled platform brain** category.
- **Narrative / the name:** the Ship-of-Theseus paradox — combat microservices update every 6 months, sensors swap, a module is battle-damaged and isolated, and the ship's coherent sense of itself *persists*. Theseus is the layer that makes the ship still the ship as every plank is replaced.
- **Boldest moves:** name the category out loud; make the tamper-evident record the *certifiable trust layer* for edge autonomy; build the fleet-mesh learning loop early (even simulated); ride the ICS cadence as distribution; readiness wedge for the money, DDIL wedge for the vision; make graceful degradation the demo hero.

## ♟ THE STRATEGIST — demand + moat + sequence
- **Demand pull (ranked):** #1 **NAVSEA SBIR NV063** (named, funded, customer-on-team, fastest validation) → #2 **ICS containerized cadence** (the insertion mechanism) → #3 readiness crisis (air cover, funds the category) → CANES/Overmatch (plumbing) → prime subcontracts (after independent proof) → allied navies (later).
- **Moat (ranked by durability):** #1 **the explainable, tamper-evident record as accreditation evidence** (the Navy's blocker is trust+ATO, not accuracy — own the evidence substrate and every algorithm must emit into it to be trusted → infrastructure). #2 DDIL-native close-the-loop-locally. #3 the NAVSEA/NIWC team (a *person* moat — a clock, convert to product fast). The ML itself is **commoditized — do not anchor the story there.**
- **Build first:** the **Local Anomaly Cell** — AIS+ADS-B+radar fusion → cold-start PoL → explainable alert + recommended action → tamper-evident record → zero shore dependency. Demos at Warhacker, maps to NV063, seeds the platform (each new algorithm a tenant on the substrate).
- **Partner vs compete:** partner for plumbing (Defense Unicorns deploy, Fathom5/ERM v4 host → back-door onto a hull, Edgerunner NL interface); **own the trust substrate + local loop**; avoid Palantir head-on (own the disconnected-ship seam); win NV063 independently *then* let primes sub you.
- **0–6–18:** Wedge (cell + record + demo + NV063 + sponsor hunt) → Beachhead (ERM v4 host + ICS cert + NV061 as 2nd tenant + ATO-evidence artifacts) → Platform (open the substrate to 3rd-party algos that emit into the record).
- **Note:** "The Navy doesn't have an accuracy problem, it has a trust-and-accreditation problem. Build the evidence layer like it's the whole company — because it is."

## 🛡 THE RED TEAM — what kills it (corrections taken)
- **Top kill-risk #1 — data access past the primes.** L3Harris/Northrop own IPMS; Lockheed/RTX own combat systems. "Fuse ship data" assumes you can *get* it — the interface is a prime-controlled ICD; approval is 12–24mo and they're incentivized to slow-walk. **Falsifier:** name the specific v1 data sources + the contractual path. Honest answer is likely "SWAN-side machinery telemetry on an open-ish network" = a smaller product than "ship brain."
- **#2 — no program-office sponsor owns the ATO risk** → a clean bundle sits in a lab forever. *Single most likely cause of death.* SBIR Phase II ≠ ATO.
- **#3 — cold-start anomaly false-alarms itself out of the wardroom.** Unsupervised, no baseline, huge operating envelope → nuisance alerts → console muted permanently (trust is one-shot). **Falsifier:** a false-alarm number on real machinery data at <~1 nuisance/watch.
- **Claims to DROP:** "self-controlled ship brain" → "onboard decision-support, human-in-command"; "fuses ALL systems" → "available SWAN-side data"; "detects anomalies (cold)" → "known fault signatures + flagged deviations for review"; "SBIR topic = demand" → "validated interest; PoR sponsorship is the open question"; "deploys in one bundle (= fielded)" → "deployable; ATO is the gate"; Pi-cluster as a *deployed-warship* claim → name ruggedized HW.
- **Feature-not-company risk** on 3 sides: Fathom5 (onboard PdM), DECK (collect→shore), Palantir/prime (fusion as a contract line). **The defensible sliver = airgap one-bundle deploy + tamper-evident record under DDIL** (the deployment & provenance substrate), not the analytics.
- **Honest pitch:** "An airgapped, tamper-evident onboard decision-support substrate (SWAN-side; combat out of scope) that deploys from one signed bundle to disconnected hulls, runs human-in-command analytics, flags known signatures + deviations for review, and keeps a verifiable record that survives DDIL — integrating alongside Fathom5/DECK, not replacing them."
- **De-risk to prove:** (1) signed ICD for one real feed; (2) a named program-office sponsor; (3) a false-alarm number; (4) the actual ruggedized HW + bounded workload; (5) evidence the substrate is hard enough a prime buys/teams rather than rebuilds.
