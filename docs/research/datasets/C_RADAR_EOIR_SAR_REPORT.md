# THESEUS Dataset Survey — Category C: Marine Radar / Sea-Clutter + EO-IR Maritime Imagery + SAR Ship Detection + Maritime MOT

**Scope:** Open-source / open-access datasets for the radar / imaging-sensor side of THESEUS (airgapped maritime onboard decision-support; AIS+ADS-B+RADAR fusion + cold-start anomaly flagging; SBIR NV063 radar fusion + explainable anomaly alerting around Navy ships; hard problem = AIS↔radar coregistration when sensors disagree).

**Methodology / honesty note:** Every URL below was hit live (FetchUrl) or surfaced via web search and cross-checked against the curated `jasonmanesis/Satellite-Imagery-Datasets-Containing-Ships` index (MIT-licensed list; 593★). License/commercial-use status was read directly from the dataset's own page **where reachable**; where the dataset page does not state a license (true for most academic Chinese SAR sets), I mark **License UNSTATED → treat as research-only / commercial UNCLEAR** and flag it LOUDLY. "Verified?" = did I confirm the landing page resolves and describes the dataset. I did **not** download any dataset or read every GitHub `LICENSE` blob; those are flagged where it matters for product/SBIR use.

## Access mechanics — top datasets (3–6 sentences each)

**xView3-SAR.** Go to `https://iuu.xview.us/`, register/login, and download the train/validation/public-test splits plus the dataset whitepaper; data is hosted for free under the xView challenge terms. Imagery is Sentinel-1 IW SAR with co-registered bathymetry and wind layers, and labels are derived by fusing **AIS tracks + automated SAR analysis + human review** — so the dataset itself is a worked example of AIS↔SAR association, including the "dark vessel" (AIS-absent) class. The main gotcha for THESEUS is licensing: it is "free and open" for the challenge but the T&C should be read for commercial/SBIR redistribution (xView-family data has historically carried non-commercial CC terms — verify before baking into a product deliverable). It is spaceborne SAR, not shipboard marine radar, so use it for the anomaly/coregistration *learning signal*, not as a sensor-domain twin.

**OpenSARShip + FUSAR-Ship (the AIS-matchup pair).** Both are accessed through Chinese academic portals (SJTU `opensar.sjtu.edu.cn` for OpenSARShip; Fudan EMW Lab / `radars.ac.cn` for FUSAR-Ship), usually after a short registration or agreement click-through. Their distinguishing value is that **every SAR ship chip is paired with its AIS message** (MMSI, length, type, look-angle), which is precisely the labeled AIS↔radar correspondence THESEUS needs to study disagreement/coregistration. Both are chip-level (cropped vessels), not full-scene tracking, and neither posts an explicit commercial license — treat as research-only until cleared. FUSAR-Ship adds fine-grained type taxonomy (15 cat / 98 subcat) useful for explainable anomaly context.

**IPIX (McMaster, Dartmouth 1993).** Direct download from `http://soma.ece.mcmaster.ca/ipix/dartmouth/datasets.html`: clutter+target files are ~16 MB each in netCDF (raw coherent I/Q), plus pre-extracted `hi.zip`/`lo.zip` ASCII I/Q for the canonical "high/low sea state" range bins. The stated condition is research use with a request to "keep us informed" and cite the website — **this is not a commercial license**, so it is a baseline/algorithm-development asset, not product data. The target is a 1 m wire-mesh styrofoam sphere (0–6 dB target-to-clutter), making it the canonical small-target-in-sea-clutter testbed for CFAR / clutter-rejection / cold-start detection on the actual radar-signal layer. Limitations: 1993 vintage, cliff-top (not shipborne), no AIS.

**Reading "iPatch" multimodal benchmark (AIS+GPS+radar+EO/IR).** The CentAUR record at `https://centaur.reading.ac.uk/101889/` describes the single most on-target set — a real shipboard suite (AIS, GPS, radar, visible + thermal) recording acted small-fast-skiff piracy attacks, spanning detection→tracking→threat recognition. **Critical caveat:** on that page the document is "Restricted to Repository staff only" and copyright is "not checked," so there is **no confirmed open download** — obtaining the data requires contacting the authors (James Ferryman, Univ. of Reading; EC iPatch project). Treat availability as UNCONFIRMED/gated despite the perfect topical fit; budget time for an author/data-request email rather than assuming a download link.

**ViCoS USV suite (MaSTr1325 / MODD2 / MODS).** Download from `https://www.vicos.si/resources/` (per-resource pages) and `github.com/bborja/modd`; well-maintained with eval scripts and a leaderboard. These are EO USV obstacle-detection/segmentation sets (boat's-eye view) — a good analog for onboard small-object detection — **but they are typically released under CC BY-NC-SA (non-commercial)**, so flag loudly before any product/SBIR use and confirm the exact license on each resource page.

## TOP PICKS for THESEUS

1. **xView3-SAR — #1 for dark-vessel anomaly + AIS↔radar coregistration supervision.** It is the only large, defense-pedigree (DIU/GFW/USCG/NMIO), validated set whose entire labeling premise is "which radar returns have no cooperating AIS track" — i.e. THESEUS's cold-start anomaly + AIS↔radar disagreement problem, at scale (220k+ instances), with co-registered environmental layers. Use it as the primary training/benchmark signal for the anomaly model. *Action:* read the xView3 T&C for commercial/SBIR redistribution before productizing.

2. **OpenSARShip + FUSAR-Ship — #1 for explicit per-contact AIS matchup.** These give literal SAR-chip↔AIS-message pairs (MMSI/length/type), the cleanest labeled substrate for learning when AIS and the radar return agree vs. disagree (spoofing, mislabeled length, dark vessel). Pair them with xView3 for breadth + correspondence depth. *Action:* register on the CN portals; clear license for non-research use.

3. **IPIX (and, if obtainable, CSIR Fynmeet) — #1 for the actual radar-signal / sea-clutter layer.** THESEUS owns a live shipboard radar feed; IPIX is the canonical open coherent I/Q small-target-in-clutter set for CFAR/clutter-rejection/cold-start baselines. CSIR Fynmeet (small-boat-in-clutter, low grazing angle) is the better domain match but is **not openly downloadable** — request from CSIR.

4. **Reading iPatch multimodal — highest-fidelity coregistration fit, BUT access UNCONFIRMED.** If the data can be obtained from the authors, it is the only open-literature set fusing AIS+GPS+radar+EO/IR on a real platform against small-fast-skiff threats — exactly NV063 around-the-ship anomaly alerting. *Action:* email Ferryman/Reading; do not assume availability.

5. **SMD + SeaShips + ViCoS USV — EO/IR complement** for the camera channel and vessel-class priors that make anomaly alerts *explainable*; all research-grade, with ViCoS being non-commercial and MASATI strictly non-commercial (exclude from product data).

## Access / gotchas note (LICENSE-FIRST)

- **Commercial-use RED FLAGS (do NOT ship in a product/SBIR deliverable without clearing):** **MASATI** (explicit non-profit research/educational only + Bing Maps terms — hard no); **ViCoS MaSTr1325/MODD/MODS** (CC BY-NC-SA non-commercial likely); **xView3-SAR** (free for challenge, but verify T&C — xView family historically CC BY-NC-SA).
- **License UNSTATED = treat as research-only / commercial UNCLEAR:** OpenSARShip, FUSAR-Ship, SSDD, LS-SSDD, HRSID, SAR-Ship-Dataset, AIR-SARShip. Most academic (esp. Chinese) SAR sets post **no license file**; absence of a permissive license means no commercial grant — clear with authors before product use.
- **Research-only by stated terms (not commercial):** IPIX (cite-website + keep-us-informed condition).
- **Access UNCONFIRMED / gated (flagged loudly):** Reading iPatch (repository PDF restricted, copyright unchecked — author request required); CSIR Fynmeet (no open portal found — request from CSIR). Both labeled UNVERIFIED as open-access.
- **Live-fetch status:** verified-live this run — iuu.xview.us, iuii.ua.es/MASATI, centaur.reading.ac.uk/101889, soma.ece.mcmaster.ca IPIX, jasonmanesis index, ViCoS/GitHub landing pages. **Not re-fetched live this run (cited via index/search, marked UNVERIFIED live):** SJTU OpenSARShip portal, AIRCAS AIR-SARShip portal.
- **Domain-gap reminder:** xView3 / all SAR sets are **spaceborne** SAR; THESEUS's radar is **shipborne marine radar**. IPIX/CSIR/MOANA are the only open *radar-signal-domain* analogs. Use SAR sets for anomaly/coregistration *logic*, IPIX/CSIR/MOANA for the radar-signal layer, and EO/IR sets for the explainability/camera channel.
- **Curated index to mine further:** `https://github.com/jasonmanesis/Satellite-Imagery-Datasets-Containing-Ships` (MIT) lists 13 SAR + 16 optical sets with per-dataset specs and direct links (incl. DSSDD, SRSDD-v1.0, S2-SHIPS, HRSC2016, Airbus Ship Detection / Kaggle).
