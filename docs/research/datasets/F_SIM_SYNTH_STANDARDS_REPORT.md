# THESEUS — Simulators, Synthetic Generators, Gov Open-Data Portals & Standards
*Category F. Verified Jun 17 2026 via live FetchUrl + web search. OPEN-SOURCE / OPEN-ACCESS only. LICENSE-FIRST: every row flags commercial-use suitability (product + SBIR). This file is the SIM / SYNTH / FEED-PORTAL / STANDARD layer; the labeled-dataset layer lives in `DATASETS.md` (don't duplicate — feeds already there are cross-referenced, not re-listed in depth).*

THESEUS context: airgapped, tamper-evident, DDIL ship-brain with **cold-start (no large historical DB)**. So the load-bearing resources are: (a) **own-data capture tooling** (cheap SDR → real AIS/ADS-B THESEUS generates itself, on representative hardware — also the Warhacker-demo money shot); (b) **simulators + synthetic generators** to augment thin/novel regions; (c) **standards corpora** for MOSA/UMAA/DDS integration credibility. Cloud feeds (Copernicus, GFW, aisstream) are **dev/training-time only** — they cannot reach an airgapped ship at sea, so they seed the cold-start, they are not a runtime dependency.

## Access mechanics — top resources (3–6 sentences each)

**AIS-catcher (jvde-github) — TOP own-AIS-capture pick.** `git clone` + `make` (or prebuilt Pi/Docker images); supports the cheapest RTL-SDR dongles *and* better SDRs (Airspy/HackRF/SDRplay) so you can scale receive quality at the venue. Outputs decoded AIS as NMEA, JSON, or UDP and ships a built-in web map, so you can demo live ship plots within minutes of plugging in an antenna. GPL-3.0 is copyleft, but THESEUS consumes its socket/JSON output as a **separate process**, so there is no license infection of THESEUS' own (proprietary) code — keep it as a sidecar binary, never statically linked. This is the single best way to satisfy "THESEUS generates its OWN real data on representative hardware," and it runs headless on a Pi for the airgapped story.

**rtl-ais (dgiardini) — simplest own-AIS-capture fallback.** Fetched and confirmed: decodes AIS from a ~$30 RTL-SDR DVB-T dongle, emits AIVDM/AIVDO over TCP/UDP (default port 10110), with official Docker images for amd64/arm32/arm64 (all Raspberry Pi flavors). One command (`rtl_ais -n`) prints live decoded packets; you must supply the dongle-specific `-p ppm` calibration for it to lock. Smaller feature set than AIS-catcher but dead-simple and battle-tested — good as a redundant second receiver. Same GPL standalone-process discipline applies.

**MSS — Marine Systems Simulator (NTNU/Fossen) — TOP synthetic-kinematics pick.** Fetched and confirmed **MIT licensed** (fully commercial- and SBIR-safe) and actively maintained (commits June 2026). It is a MATLAB/Simulink library that also runs in **free GNU Octave**, with validated 6-DOF dynamics for ships, USVs, AUVs and floating structures plus GNC, INS/Kalman, and 1st/2nd-order wave-load models. For THESEUS cold-start it generates physically-realistic vessel trajectories and sea-state-perturbed motion to augment thin/novel maneuver regimes that no historical AIS covers. Caveat: building *new* hull hydro models needs a ShipX or WAMIT license, but the bundled craft models simulate freely.

**Stone Soup (UK DSTL) — TOP fusion-synthesis pick.** MIT-licensed Python framework purpose-built for tracking and **multi-sensor data fusion**, including first-class **ground-truth simulators and synthetic detection generators**. This directly attacks THESEUS' weakest data gap (no open AIS+radar+ADS-B fusion set, per DATASETS.md): you script a ground-truth scenario, synthesize radar/EO/AIS detections with controllable noise/clutter, and evaluate cold-start track-fusion before any real data exists. National-lab provenance (DSTL) is a credibility asset for SBIR. It's a framework, so you author the scenarios.

**UMAA + OpenDDS — the integration-standard spine.** UMAA (US Navy PEO USC) is the authoritative unmanned-maritime autonomy interface standard; its Distribution-A Interface Control Documents (Situational Awareness ICD, EO ICD, Engineering Ops ICD, etc.) and IDL are publicly downloadable from AUVSI, while the full 6.0 SDK is Distro-D (free access request). UMAA interfaces ride on **OMG DDS**, and **OpenDDS** is a permissively-licensed (non-copyleft, commercial-OK) open-source DDS implementation you can use both to implement those interfaces and to generate sample DDS traffic for testing. Together they let THESEUS claim concrete MOSA-aligned, Navy-standard interoperability rather than a bespoke bus. Pair with the public-domain Naval-MOSA guidebook for the proposal narrative.

**Copernicus (CMEMS + Data Space / Sentinel-1) — environmental + SAR seed.** Both are free under the Copernicus licence with commercial use permitted (attribution required) after a free registration; CMEMS exposes a Python toolbox/API for ocean physics (currents, SST, waves, sea-ice) and the Data Space Ecosystem serves Sentinel-1 SAR via S3/OData. SAR is the key augmentation for dark-vessel/radar training where AIS is silent. **Critical DDIL caveat:** these are cloud feeds — they seed the cold-start model **on shore at build time**; they cannot reach an airgapped ship at sea, so they are a training-time input, never a runtime dependency.

## TOP PICKS for THESEUS

1. **AIS-catcher + rtl-ais (own real AIS) and dump1090-fa (own real ADS-B)** — the Warhacker-demo centerpiece: a ~$30 RTL-SDR on a Raspberry Pi has THESEUS **generating its own real, live maritime + air picture on representative airgapped hardware**, no cloud, no historical DB. This *is* the cold-start story made physical. Capture is receive-only and legal (NAVCEN: only *transmitting* AIS base stations are restricted).
2. **MSS (MIT)** — commercial-safe synthetic vessel kinematics + sea-state augmentation to fill thin/novel regions; runs in free Octave on the edge.
3. **Stone Soup (MIT)** — synthesize multi-sensor detections + ground truth to build/evaluate the radar+AIS+ADS-B fusion organ that has no open real dataset.
4. **UMAA (Distro A) + OpenDDS (permissive)** — concrete Navy-standard, MOSA-aligned integration spine; pure credibility + interoperability for SBIR/program.
5. **pyais (MIT)** — glue: decode captured NMEA into structured tracks for THESEUS ingest.

## Access / gotchas

- **License discipline:** GPL tools (AIS-catcher, rtl-ais, MOOS-IvP) are fine **as standalone sidecar processes** consumed over sockets — never statically link into THESEUS' proprietary core. **MIT/Apache** picks (MSS, Stone Soup, pyais, AISonobuoy, OpenDDS) are clean for product + SBIR.
- **LOUD flag — Global Fishing Watch is NON-COMMERCIAL only.** Excellent dark-vessel labels, but per GFW's own FAQ the APIs are free "for non-commercial purposes." Use strictly for **method validation / eval**, never as a component of a product or SBIR deliverable.
- **Flag for legal — MOOS-IvP (GPLv3 + historical academic-use nuance) and TrAISformer (research code, license unclear).** Treat both as *reference/method* sources; verify current LICENSE before any product or SBIR integration.
- **DDIL reality:** every cloud feed (Copernicus, GFW, OpenSky, aisstream) is **shore/build-time seed only** — none can reach an airgapped ship. The only runtime-legal data path at sea is **own SDR capture** (picks #1) + onboard sensors. Design THESEUS so cloud feeds train the cold-start model ashore and own-capture sustains it underway.
- **No public realtime bulk AIS from US gov:** NAVCEN/NAIS is operational (no public live download); historical bulk is NOAA MarineCadastre (see DATASETS.md). Live realtime = capture-your-own or (dev-only) aisstream.io.
- **UMAA 6.0 latest SDK is Distro D** (free access request via AUVSI); Distro-A ICDs/IDL are directly downloadable now — start there.
- **COLREGs:** use the **US Navigation Rules (33 CFR, public domain)** or consolidated government PDFs for an open machine-readable corpus; the IMO consolidated edition (IC904E) is a paid, copyrighted publication.
- **Verify-before-proposal:** exact LICENSE files for AIS-catcher, dump1090 fork, pyais, AISonobuoy, OpenDDS, and TrAISformer should be re-read at proposal time — labels here are from repo metadata + project pages (UNVERIFIED exact-text where noted "verify").
